# Use a semantic Darkroom Design System across all in-app UI

Prokudin will modernise every owned in-app surface through one token-based Darkroom Design System rather than isolated per-view restyling. It preserves the Restoration Workspace structure while using a compact, neutral, image-led dark default, a single interaction accent, RGB only for channel identity, accessible state cues, and shared components; operating-system file dialogs remain native.
